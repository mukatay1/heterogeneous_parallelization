1. Программа разбивает массив на фрагменты и распределяет их между блоками потоков GPU, которые параллельно сортируют свои части данных и затем пошагово сливают их в единый упорядоченный массив

2. Программа разделяет массив на сегменты и распределяет их между множеством потоков GPU, где каждый поток независимо выполняет быструю сортировку своего участка по опорному элементу. Такой подход позволяет одновременно упорядочивать разные части данных, используя тысячи ядер видеокарты для сокращения общего времени обработки.

3. Программа переносит процесс построения и перестройки бинарной кучи на видеокарту, где параллельные потоки одновременно восстанавливают свойства кучи для разных узлов дерева.

4. Для сравнения производительности мы запускаем одинаковые алгоритмы на центральном процессоре и видеокарте, замеряя время их работы с помощью высокоточных таймеров. Это позволяет наглядно увидеть точку перелома, когда скорость вычислений на GPU начинает оправдывать затраты времени на пересылку данных из оперативной памяти в видеопамять.

Контрольные вопросы:

1) Последовательная сортировка слиянием выполняется одним потоком на одном ядре процессора по очереди, в то время как параллельная версия одновременно обрабатывает разные части массива на множестве ядер или потоков.

2) Количество потоков и блоков в CUDA определяет уровень загрузки графического процессора, где оптимальное распределение позволяет максимально задействовать все вычислительные модули и скрыть задержки при доступе к памяти.

3) Реализация быстрой сортировки на GPU осложняется рекурсивной природой алгоритма и неравномерной загрузкой ядер из-за разного размера разделяемых частей массива после каждого шага.

4) Параллельная сортировка на GPU проигрывает версии для CPU на малых массивах, так как время на копирование данных между оперативной памятью и видеокартой превышает выгоду от ускорения самих вычислений.

5) Правильный выбор размера блоков и потоков критичен для того, чтобы избежать простоя оборудования и эффективно использовать ограниченные ресурсы каждого мультипроцессора, такие как регистры и общая память.

6) Разделяемая память значительно ускоряет сортировку, так как позволяет потокам внутри одного блока обмениваться данными намного быстрее, чем через основную медленную видеопамять.

7) Принцип разделяй и властвуй заключается в рекурсивном разбиении задачи на мелкие подзадачи до тех пор, пока они не станут элементарными, с последующим объединением их результатов в общее решение.
