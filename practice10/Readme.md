Practice 10

Задание 1. Анализ производительности CPU-параллельной программы (OpenMP)

В ходе работы была реализована параллельная программа на C++ с использованием OpenMP для вычисления статистик массива данных. Выполнено измерение времени работы при разном числе потоков и проведён анализ ускорения с точки зрения закона Амдала.

Задание 2. Оптимизация доступа к памяти на GPU (CUDA)

Были реализованы две версии CUDA-ядра с коалесцированным и некоалесцированным доступом к глобальной памяти. Проведено измерение времени выполнения и показано влияние оптимизации доступа к памяти и использования разделяемой памяти на производительность GPU.

Задание 3. Профилирование гибридного приложения CPU + GPU

Разработано гибридное приложение, в котором вычисления распределены между CPU и GPU с использованием CUDA streams и асинхронных копирований данных. Выполнено профилирование, выявлены накладные расходы передачи данных и реализована оптимизация для их уменьшения.

Задание 4. Анализ масштабируемости распределённой программы (MPI)

Реализована MPI-программа для вычисления агрегатных функций над массивом данных. Исследованы strong scaling и weak scaling, а также влияние коммуникационных операций на масштабируемость алгоритма.

Контрольные вопросы

1. В чём отличие измерения времени выполнения от профилирования?
Измерение времени показывает общее время работы программы, тогда как профилирование позволяет определить, какие части кода потребляют больше всего времени и ресурсов.

2. Какие виды узких мест характерны для CPU, GPU и распределённых программ?
Для CPU характерны ограничения по числу ядер и памяти, для GPU — пропускная способность памяти и неэффективный доступ к ней, а для распределённых программ — задержки и накладные расходы на коммуникации.

3. Почему увеличение числа потоков или процессов не всегда приводит к ускорению?
Из-за наличия последовательных участков кода, накладных расходов на синхронизацию и ограничений памяти ускорение может снижаться или полностью исчезать.

4. Как законы Амдала и Густафсона применяются при анализе масштабируемости?
Закон Амдала оценивает предел ускорения при фиксированном размере задачи, а закон Густафсона показывает масштабируемость при увеличении объёма вычислений.

5. Какие факторы наиболее критичны для производительности гибридных приложений?
Ключевыми факторами являются скорость передачи данных между CPU и GPU, перекрытие вычислений и обменов, а также баланс нагрузки между вычислительными устройствами.
