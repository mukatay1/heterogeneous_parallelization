Задача 1.
В первой задаче я реализовал параллельный алгоритм редукции на GPU с использованием CUDA для вычисления суммы элементов массива, применяя разделяемую память для оптимизации.

Задача 2.
Во второй задаче я реализовал алгоритм сканирования (префиксной суммы) на GPU, используя разделяемую память и проверив корректность результатов на тестовых данных.

Задача 3.
Также был проведён анализ производительности редукции и сканирования для массивов разного размера и выполнено сравнение скорости работы GPU и CPU реализаций.

Контрольные вопросы:

Разница между редукцией и сканированием заключается в том, что редукция сводит массив к одному значению, а сканирование вычисляет накопленный результат для каждого элемента.

Для оптимизации редукции и сканирования используются глобальная и разделяемая память CUDA, так как разделяемая память обеспечивает более быстрый доступ внутри блока.

Оптимизация префиксной суммы на GPU достигается за счёт использования разделяемой памяти и минимизации обращений к глобальной памяти.

Сканирование применяется, например, в задачах сортировки, построения гистограмм и алгоритмах параллельной обработки данных.
