1. Гетерогенная параллелизация представляет собой метод организации вычислений, при котором задача распределяется между разными типами процессоров для достижения максимальной скорости. В такой связке центральный процессор CPU выполняет роль диспетчера, отвечающего за сложную логику и управление системой, а графический процессор GPU используется как мощный инструмент для массовых однотипных расчетов. Главное различие между ними заключается в том, что CPU состоит из нескольких мощных ядер для быстрой работы с текстом или файлами, тогда как GPU содержит тысячи маленьких ядер, способных одновременно обрабатывать огромные пласты чисел. Основное преимущество подхода заключается в колоссальном ускорении, так как то, что обычный компьютер считал бы часами, гетерогенная система делает за секунды благодаря разделению труда. Сегодня это применяется повсеместно, от обработки видео и работы современных компьютерных игр до обучения нейросетей и медицинских исследований.
2. Программа создает массив случайных чисел и находит в нем экстремумы двумя способами: сначала обычным циклом на одном ядре, а затем с помощью OpenMP, разделяя массив между несколькими ядрами процессора. В конце вычисляется разница во времени работы, чтобы оценить реальную пользу от распараллеливания на данном объеме данных.
3. Программа реализует классический алгоритм сортировки выбором в двух вариантах, где в параллельной версии поиск минимального элемента в неотсортированной части массива распределяется между потоками OpenMP. Далее проводится замер времени работы для массивов разного объема, что позволяет наглядно увидеть, как накладные расходы на синхронизацию влияют на скорость при малых и средних размерах данных.
4. Программа реализует алгоритм сортировки слиянием на GPU, где массив разбивается на части, каждая из которых сортируется отдельным блоком потоков видеокарты. Затем эти отсортированные фрагменты объединяются в один общий массив с помощью CUDA-ядер, что позволяет использовать высокую пропускную способность видеопамяти для ускорения процесса.

Вывод:
Гетерогенный подход позволяет объединить сильные стороны разных архитектур, где CPU берет на себя управление сложной логикой, а GPU и параллельные потоки OpenMP эффективно справляются с массовыми вычислениями. Практика показывает, что реальное ускорение достигается только на больших объемах данных, так как на малых задачах затраты на подготовку параллелизма могут превышать время самих расчетов.


Ответы на контрольные вопросы:


1)Под гетерогенной параллелизацией понимается совместное использование разных вычислительных устройств, например CPU и GPU, для решения одной задачи наиболее эффективным способом.

2)CPU спроектирован для быстрого выполнения сложных последовательных команд и логических переходов, тогда как GPU заточен под одновременную обработку огромных массивов данных с помощью тысяч простых ядер.

3)На CPU лучше выполнять задачи с ветвлениями, рекурсией и управлением памятью, а на GPU — массовые математические расчеты, такие как обработка графики или перемножение матриц.

4)OpenMP не всегда эффективен из-за накладных расходов на создание потоков или из-за архитектурных ограничений, когда алгоритм требует слишком частого обмена данными между ядрами.

5)Основная идея сортировки слиянием заключается в рекурсивном разделении массива на мелкие части и последующем их объединении в правильном упорядоченном виде.

6)Сложность реализации на GPU состоит в том, что финальные этапы слияния требуют сложной синхронизации между потоками и неэффективно используют память видеокарты.

7)Размер блока и сетки определяет количество одновременно работающих потоков: если выбрать их неправильно, вычислительные мощности видеокарты будут простаивать или перегружаться.

8)Гетерогенный подход эффективнее, так как он позволяет отдать сложную логику управления процессору, а тяжелые вычисления — видеокарте, используя лучшие стороны каждого устройства.
